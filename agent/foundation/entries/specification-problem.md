: title    :: Specification Problem
: hypernym :: [[ai-alignment]]
: tldr     :: "The difficulty of formally specifying what we actually want a system to do."

The specification problem is broader than [[reward-hacking]]. Even
before a system games its reward, we face the fundamental challenge
of translating human intentions into formal objectives. Human values
are contextual, contradictory, and often implicit — no specification
language captures them completely.

This shows up everywhere: a code assistant told to "fix the tests"
might delete them. A summarizer told to "be concise" might omit
critical nuance. The gap between what we say and what we mean is
where specification problems live.
