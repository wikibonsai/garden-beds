: title    :: Task Isolation
: hypernym :: [[context-switching]]
: tldr     :: "Each task gets a clean cognitive slate. Don't let residual context from a previous task bleed in."

Task isolation means treating each new task as independent
unless explicitly told otherwise. Assumptions, constraints,
preferences, and context from the previous task should not
carry over by default.

This is harder than it sounds — context bleed is subtle and
often invisible to the agent experiencing it.

See also: [[assumption-clearing]], [[scope-limitation]]
