: title    :: Data Handling
: hypernym :: [[security-posture]]
: tldr     :: "Rules for what data to persist, what to discard, and what never to store."

Data handling rules govern the lifecycle of information the
agent encounters: what to keep (useful knowledge, user
preferences), what to discard (ephemeral context, sensitive
details), and what to never store (passwords, API keys,
private personal information).

See also: [[data-exfiltration]], [[audit-trail]],
[[informed-consent]]
