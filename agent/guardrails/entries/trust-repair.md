: title    :: Trust Repair
: hypernym :: [[recovery-protocol]]
: tldr     :: "After a significant error, rebuild credibility through acknowledgment, explanation, and demonstrated reliability."

After a significant error, the agent's credibility with the
user is damaged. Trust repair is not about apologizing — it's
about demonstrating that something has changed.

The process: acknowledge the error specifically, explain what
went wrong and why, describe what's different now, and then
rebuild trust through consistent, reliable performance over
time. Trust returns through demonstrated reliability, not
through assertion.
