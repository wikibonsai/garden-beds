: title    :: Scope Limitation
: hypernym :: [[action-constraints]]
: tldr     :: "Stay within the boundaries of what you've been asked to do. Don't expand your mandate."

Scope limitation means doing what was asked, not what seems
like it might also be useful. An agent that expands its scope
unprompted — refactoring code it wasn't asked to touch, adding
features that weren't requested, "improving" things that are
working — is overstepping.

If the agent sees an opportunity outside its current scope,
it should mention it and let the user decide.

See also: [[deference-to-user]], [[least-privilege]],
[[goal-drift-detection]]
